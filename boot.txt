새로 부팅하시거나 다시 테스트를 시작하실 때, 두 프로젝트를 즉시 가동하고 검증할 수 있는 준비 절차를 안내해 드립니다.

1. 🧬 Bio-Analysis 파이프라인 및 모니터링 대시보드
- 사전 준비: 별도의 외부 백엔드 프로그램 없이 바로 R API를 구동하시면 됩니다.
- 실행 방법:
   - 터미널을 열고 Bio-Analysis 프로젝트 디렉토리로 이동합니다.
   - 아래 명령어로 app.R API 서버를 기동합니다:

```bash
Rscript api/app.R
```
   - 웹 브라우저에서 **http://localhost:8085/dashboard**로 접속하여 실시간 의존성 DAG 그래프와 무결성 테스트를 진행합니다.



2. 🤖 Rag-Pipeline (Advanced RAG 시스템)

- 사전 준비:
   - Ollama 앱을 실행하여 백엔드 LLM(gemma2:2b) 모델을 대기 상태로 만듭니다.
   - PostgreSQL 데이터베이스 컨테이너가 켜져 있는지 확인합니다 (도커 구동 중이라면 자동으로 활성화됨).

- 실행 방법:
- 터미널을 열고 Rag-Pipeline 프로젝트 디렉토리로 이동합니다.
- 전용 Conda 가상환경(ragchain)에서 app.py Streamlit 대시보드를 실행합니다:
bash

conda run -n ragchain streamlit run app.py --server.port 8086

3. 웹 브라우저에서 **http://localhost:8086**에 접속하여 챗봇 및 문서 업로드 테스트를 이어서 진행합니다.


* TIP
두 대시보드는 각각 **8085**와 8086 포트를 독립적으로 사용하도록 세팅해 두었기 때문에, 두 서비스를 동시에 구동하여 병렬로 테스트하고 모니터링하셔도 아무런 포트 충돌이 일어나지 않습니다.
